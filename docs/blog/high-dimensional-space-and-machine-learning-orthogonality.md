# 高维空间与机器学习（二）：为什么高维向量几乎总是正交？

上一篇文章讨论了高维空间里“距离失效”的问题。一个自然的后续问题是：如果距离越来越不可靠，那么高维空间里到底还有什么几何结构是可靠的？

一个非常重要的答案是：角度。更准确地说，高维随机向量之间的夹角会高度集中在 90 度附近。换句话说，两个随机向量在高维里几乎总是近似正交的。

第一次听到这句话时，很多人会觉得难以接受。二维空间里，两条随机方向大多数时候并不正交；三维里也一样。为什么维度一高，正交反而变成常态？而且，这件事为什么和机器学习、尤其是 embedding 空间如此相关？

这篇文章想从实验、几何和表示学习三个层面，解释这个现象。

## 1. 从随机向量实验开始：cosine 为什么会集中到 0？

考虑两个随机向量

$$
x, y \in \mathbb{R}^d,
$$

它们的每个坐标都独立同分布，满足标准高斯或对称分布。我们关心的是它们的 cosine similarity：

$$
\cos \theta = \frac{x^\top y}{\|x\| \|y\|}.
$$

如果在低维里做实验，你会看到 cosine 分布比较宽：有的接近 1，有的接近 -1，也有不少落在中间。但当维度不断增加时，这个分布会迅速向 0 聚集。

原因并不复杂。分子 $x^\top y$ 是许多独立乘积项的和：

$$
x^\top y = \sum_{i=1}^d x_i y_i.
$$

如果各坐标独立且均值为 0，那么这些项的期望为 0，方差有限，因此总和的量级大约是 $\sqrt{d}$。而分母 $\|x\| \|y\|$ 的量级大约是

$$
\sqrt{d} \cdot \sqrt{d} = d.
$$

所以整体上

$$
\cos \theta \sim \frac{\sqrt{d}}{d} = \frac{1}{\sqrt{d}}.
$$

这意味着维度越高，cosine 的典型大小越小。更精确地说，当向量经过归一化之后，随机内积的均值接近 0，方差大约是 $1/d$。于是我们会观察到一个非常稳定的结论：

> 在高维空间中，随机向量之间的夹角几乎总在 90 度附近。

这就是所谓的 cosine distribution concentration。它并不是说所有向量都严格正交，而是说“偏离正交很多”的情形越来越少。随着维度增加，0 附近的窄峰变得越来越尖，余弦分布越来越集中。

## 2. 几何解释：高维空间为什么有这么大的容量？

这个结论之所以反直觉，是因为我们在低维里对方向的感受非常有限。在二维平面中，方向本来就只有一个圆周；你随便选几条线，很快就会挤在一起。三维也类似，球面虽然比圆大，但可容纳的“明显分开”的方向仍然有限。

但高维空间的单位球面拥有巨大的方向容量。这里的“容量”不是体积意义上的大，而是“你可以安放非常多彼此夹角接近 90 度的方向，而且它们仍然互不冲突”。

从编码角度看，这很关键。因为如果向量之间能够大规模地保持接近正交，那么它们就能在一个共享空间里彼此区分，而不容易相互干扰。高维空间并不是把点“挤在一起”的容器；相反，它提供了大量几乎互不重叠的方向自由度。

这听起来似乎和上一篇“距离集中”矛盾，但其实完全不矛盾。距离集中说的是随机点之间的长度关系会变得相似；而“近似正交”说的是方向关系会集中到 90 度附近。两者共同构成了高维空间的典型几何：

- 点大多落在某个薄壳上；
- 向量方向大多彼此近似正交；
- 因而长度差异越来越不重要，方向结构反而更值得关注。

所以高维空间真正宝贵的，不是让距离更有区分度，而是让方向组合具有极高的表达能力。

## 3. embedding 空间为什么可以容纳 100k tokens？

现在把这个几何事实和现代机器学习连接起来。一个语言模型的 embedding 表通常要容纳数万到数十万个 token。很多人会自然问：这么多 token 放进同一个向量空间里，难道不会互相挤爆吗？

如果我们还停留在低维直觉里，这个担忧很合理。但高维空间恰恰擅长处理这种问题。假设 embedding 维度是 768、1024 或 4096，这些维度已经足够大，能够容纳海量彼此差异化的方向。

当然，token embedding 并不是完全随机向量，也不需要严格正交。事实上，语义相关的词应该彼此更接近，不相关的词则应当分开。这里真正重要的是：

1. 高维空间提供了足够多的可用方向。
2. 训练过程会把这些方向组织成有语义结构的分布。
3. 即使容纳 100k tokens，也不需要每一对 token 都相隔很远，只要局部和全局结构都可区分即可。

高维的巨大容量使得“同时编码大量对象”成为可能。它像一个极大的地址空间，不同 token 可以占据不同区域和方向，而不至于频繁发生冲突。换句话说，维度不是浪费，它是在为“可分性”和“可组合性”买空间。

更进一步，embedding 不只是存 token 的身份，还要容纳语义、语法、上下文关系以及可组合性。高维空间正适合承担这种多重约束，因为它允许模型在不同子方向上编码不同类型的信息。某些方向可能与词性相关，某些方向与主题相关，某些方向与上下文兼容性相关。

如果空间太低维，这些结构会被迫强烈耦合；而在高维里，模型可以利用近似正交的方向，把不同因素更自然地拆开。

## 4. 高维近似正交，对表示学习意味着什么？

这里有一个值得强调的点：高维向量“几乎正交”并不是说 embedding 学到的东西天然有语义，而是说这个空间具有非常适合承载语义的几何条件。

随机空间只给了我们容量，学习过程才赋予它结构。也就是说：

- 高维空间告诉我们“装得下”；
- 训练目标告诉我们“怎么装”；
- 数据分布告诉我们“应该把什么放在一起”。

如果一个模型只是随机初始化却不训练，那么向量之间虽然近似正交，但没有任何语义价值。只有经过训练之后，这个空间才会演化成一个真正的编码空间：相似概念共享局部结构，不同概念占据不同方向，不同任务信号沿着不同子空间传播。

这也是为什么在理解 embedding 时，我们不能只看一对词向量之间的距离，还要看整个空间的几何组织。一个好的 embedding 空间，并不是让所有相关词都挤成一团，而是在大尺度上形成稳定的语义布局，在小尺度上保留细粒度的区分。

## 5. 我们的理解：embedding 空间本质上是一种语义编码空间

如果把这一切总结成一句话，我更愿意这样说：

> embedding 空间不是一个“装数字的表格”，而是一个高维的语义编码空间。

“近似正交”是这个编码空间的底层几何优势。它使得空间有足够多的方向自由度，让模型能同时表示大量对象、关系和上下文因素。它并不直接等于语义，但它为语义提供了容量和分离性。

这也是为什么现代表示学习如此依赖高维向量。高维不是副作用，而是条件。没有足够高的维度，我们很难期待一个空间既能表达大量概念，又能保留细粒度结构，还能支持线性变换、检索和下游任务。

所以，当我们说“高维向量几乎总是正交”时，真正重要的不是这句话本身，而是它背后的启发：高维空间之所以对机器学习有价值，不是因为它神秘，而是因为它以一种非常特殊的方式，把“容量”和“可分性”同时给了我们。
